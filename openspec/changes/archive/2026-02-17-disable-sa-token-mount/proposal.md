## Why

Workflow pods have no need to call the Kubernetes API, yet the default ServiceAccount token is mounted at `/var/run/secrets/kubernetes.io/serviceaccount/`. If a container is compromised, this token grants API access to the cluster. Setting `automountServiceAccountToken: false` at the pod spec level eliminates this attack surface.

## What Changes

- Add `automountServiceAccountToken: false` to the pod spec in the Deployment template generated by `pkg/builder/k8s.go`
- Add test assertion verifying the field is present in generated manifests

## Capabilities

### New Capabilities
- `disable-sa-token`: Disable automatic ServiceAccount token mount in generated Deployment pod specs
